数当てゲーム

コンピュータの乱数で1~10の値を一つ決めて、それを答えとする。ユーザは答えを予想し、その数を最大で3回入力できる。ユーザの入力が外れた場合は、ヒントとして入力よりも答えが大きいか小さいかを表示する。3回以内に答えを当てればユーザの勝ち、そうでなければ負け。ページにはテキストボックスとボタンが設置されているユーザはテキストボックスに予想した数を入力し、ボタンを押すと判定が出る。

数当てゲームページ:kazuate.html

各授業回の学習事項

第1回 WWW と HTML

HTMLは数々の「要素」で作られているということを学んだ。その要素にはブロック要素とインライン要素があり、インライン要素はブロック要素の内側でなければならないなどの決まりがあることを知った。また、要素の種類によってできることが異なる。例えば、リスト要素で箇条書きのようなレイアウトにすることができる。このような要素は数え切れないほどの種類を調べるにあたってMDN Web Docs上で調べることが大切だということを学んだ。

演習の成果:演習1-3:HTMLの文書の作成

演習の成果:演習1-4:HTMLのさまざまな要素

第2回 CSS による Web ページの装飾

CSSでページレイアウトを考え装飾した。テキストサイズの変更やテキストボックスを作り、そのボックスの中に画像を入れたり枠を作った。注目されるべきである一番大きな文字に対して影をつけるなどの工夫を入れることで強調した。

演習の成果browser-intro.html

第3回 JavaScript の基本

JavaScripsはインタプリタ言語でコンパイル言語のJavaとは名前が似ているが別物になっている。Javaにはコンパイルという一手間が必要だが、JavaScriptにはそれが必要ないという利点がある一方で、エラーになりづらかったりエラーに気づきづらいというデメリットもある。授業ではコンソール画面でJavaScriptで書いたコードを表示させることをした。その際のJavaとの違いは、複雑な型の指定をせずに全てletにしたり、関数の書き方がスッキリしていて見やすかったりしていて人間に見やすい言語だと思った。エラーについてはエラーにならないプログラムが存在していて、間違った式の計算結果は全てNaN(Not a Number)と表示される。これはエラーではないのでプログラムは停止することなく実行され続けてしまう。他にundefined(未定義)やnull(空っぽ)があることを学んだ。

演習の成果:obj-practice.html

第4回 DOM と JavaScript

イベント処理の仕方として、HTMLでページ上に設置するテキストボックスとボタンを作り、ボタンを押した後の処理をJavaScriptでするやり方をした。input要素は入力項目を意味している。type属性:入力する値の種類 name属性:input要素の名前 size属性:テキストボックスの横幅 label要素は入力項目を説明する働き for属性:input要素のidを指定する(label要素のfor属性とinput要素のidを同一に合わせる) buttom要素は次のp要素にメッセージを表示するのに使います。(検索する際にdocument.Selector(要素名#id名))document.Selectorで要素検索ができることを知った。検索したものを変数に入れて、textContent:テキストを参照 getAttribute(属性名):要素の属性値を参照 input要素.value:ユーザが設定した値を参照する ことができる。

演習の成果:tashizan.html

第5回 イベント処理

要素の新規作成、要素の属性を追加したり変更したり、要素の削除の仕方を学んだ。(要素の参照については ~.~で変数の代入を省略可能だが、作成は省略しない方がいい) 構文: createElement(要素名): 要素を新規作成 要素1.insertAdjacentElement('beforeend', 要素2): 要素1の内部に要素2を配置 要素1.insertAdjacentElement('afterend', 要素2): 要素1の直後に要素2を配置 要素.setAttribute(属性名, 値): 要素の属性を設定 要素.remove(): 要素を削除 などの機能がある。
